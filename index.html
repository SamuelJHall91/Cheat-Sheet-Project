Semantic HTML Tags Cheat Sheet

Document Semantic HTML

Tag Name Description
<title> Title Shown in the browser tab & search results. Should be unique for every page on the site.
<footer> Footer When inside <body> it's the website footer. When inside <article> it's the least important information.
<article> Article A peice of content that's independant. Could be removed from the website and still make sense.
<main> Main Primary content of the page
<section> Section A group series of related content pieces
<header> Header When insdie the <body> it's the website masthead. When inside <article> it's the most important information.
<nav> Nav Defines a group of navigation links.
<aside> Aside Secondary content not required to understand the main content.

Lists

Tag Name Description
<ul> Unordered List An unordered list. The order of the list isn't important. Can only have <li> elements as direct children.
<ol> Ordered List An ordered list. The order of the items is important. Could be alphabetical, numerical, best to worst, etc. Can only have <li> elements as direct children.
<li> List A single list item. Must be inside a <ul>, <ol> or <menu>. Can have most other elements inside it.
<dl> Description List A description list. A grouping of terms and definitions. Words & definitions, titles & summaries, data points, etc. Can only have <dt> and <dd> elements as direct children
<dt> Description Title Description Title, the term of the item. Must come before the <dd>.
<dd> Description Definition Description definition, the data, or text of the item. Can be multiple <dd> tags underneath one <dt>

Text

Tag Name Description
<a href="..."> Anchor Tag For making hyperlinks. href is the path to where the link should go.
<h1> Heading 1 The most important piece of content on the page. On the homepage this should be the company's name. On the inside pages this should be the page title.
<h2><h3><h4><h5><h6> Content Headings 2 - 6 Content headings, each a sub-heading of the one above.
<hgroup> heading Group Allows you to group multiple headings together and have them semantically treated as a single heading. It's primary purpose is for subheadings.
<p> Paragraph A generic paragraph of text
<blockquote> Blockquote A large, stand alone quote from another source.
<cite> Citation A citation from another source, often used with quotations. A person's name, a URL, a book, a movie title, etc.
<q> Quotation A small quotation embedded within other content.
<em> Emphasize AA string of emphasized, slightly more important text. Screen readers will change their voice for this text.
<strong> Strong A string of highly emphasized, much more important text. Screen readers will change their voice for this text.
<ins datetime="..."> Insert datetime Content that was inserted after the document was published. datetime defines when it was added.
<del datetime="..."> Delete datetime Content that was deleted after the document was published. datetime defines when it was removed.
<abbr title="..."> Acronym/Abbreviation An Acronym or abbreviation, like "HTML", "CSS", etc. title contains the expended version.
<dfn> Definition A definition of a term on the page. Should only be used once of the term.
<mark> Mark Used to highlight a piece of text for reference. The keywords in a search results page, the current navigation item.
<i> I Defines technical term, a ship name, a book title, a thought, sarcasm, another language.
<b> B Defines a keyword, like product name in a review, a lead sentnece in a paragraph.
<s> S Content that's no longer relevant to the document. Consider if the <del> element is beter suited first.
<u> U Labels the text as having a non-textual annmotation. A misspelled word, a Chinese proper name, etc.
<small> Small Represents side comments and fine print.
<address> Address Contact information, email, tel, postal address, etc.

Images & Media

Tag Name Description
<img src="..."> Image Embeds an image that's important to the content. src is a path to the image file. alt describes the image if it cannot be seen.
<figure> Figure Embeds annotated images, illustrations, photos, code, etc. Could be moved out of place and would still make senese.
<figcaption> Figcaption For adding a caption/annotation to the <figure>. Must be inside a <figure> element. Cannot stand alone.
<img src="..."> Image Embeds an image that's important to the content. src is a path to the image file. alt describes the image if it cannot be seen.
<picture> Picture Respnsive image insertion. Allows developers to provide different images for different contexts.
<video poster="..." autoplay loop muted controls> Video Poster For embedding movies into a website. poster is the path to an image that's displayed before the video plays. autoplay will hint the video to start automatically. loop triggers whether the video should repeat or not. muted can be added to not play sound by default. controls shows or hides the browser's player buttons.
<audio autoplay loop muted controls> Audio For embedding sounds into a website. autoplay will hint the audio to start automatically. loop triggers whether the audio should repeat or not. muted can be added to not play sound by default. controls shows or hides the browser's player buttons.
<source> Source Must be inside <picture>, <video> or <audio> to define the different versions of content. For example, in video it gives paths to the MP4 and WEBM formats.
<track> Track Used to pair captions, chapters, etc. with <video> elements.
